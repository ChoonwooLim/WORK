# 📘 RemoteAGT 사용 매뉴얼

> AI 원격 지휘·통제 시스템 — 사용자 가이드
>
> 이 매뉴얼은 RemoteAGT 대시보드와 텔레그램 봇의 모든 기능을 상세히 안내합니다.

---

## 1. 시작하기

### 1.1 로그인

1. 브라우저에서 RemoteAGT 대시보드 주소에 접속합니다
2. **이메일**과 **비밀번호**를 입력하고 **로그인** 버튼을 클릭합니다
3. 로그인에 성공하면 자동으로 대시보드 메인 화면으로 이동합니다

> 비밀번호를 잊으셨나요? 관리자에게 비밀번호 초기화를 요청하세요.

### 1.2 회원가입

1. 로그인 페이지에서 **회원가입** 탭을 클릭합니다
2. **사용자명**, **이메일**, **비밀번호**(6자 이상)를 입력합니다
3. 가입 완료 후 자동으로 대시보드에 접속됩니다

> 가입 후 관리자가 역할(권한)을 설정하면 더 많은 기능을 사용할 수 있습니다.

### 1.3 로그아웃

사이드바 하단의 **⏏ 로그아웃** 버튼을 클릭합니다.

## 3. 페이지별 상세 기능

### 3.1 📊 대시보드 (메인 화면)

대시보드는 시스템의 전체 상태를 실시간으로 보여주는 메인 화면입니다.

**시스템 메트릭 카드 (상단 4개):**

| 카드 | 표시 정보 | 색상 변화 |
|------|----------|----------|
| 🖥️ **CPU** | 현재 CPU 사용률 (%) + 막대 그래프 | 80% 이상 시 빨간색 |
| 💾 **메모리** | RAM 사용률 (%) + 막대 그래프 | 85% 이상 시 빨간색 |
| 📀 **디스크** | 저장공간 사용률 (%) + 막대 그래프 | 85% 이상 시 빨간색 |
| 🐳 **컨테이너** | 실행 중 / 전체 컨테이너 수 | — |

> 메트릭은 **30초마다** 자동으로 갱신됩니다.

**Orbitron 컨테이너 현황:**
- 현재 Orbitron으로 배포된 모든 프로젝트의 Docker 컨테이너 상태
- 각 카드에는 **이름**, **이미지**, **포트**, **상태**가 표시됩니다
- ✅ 실행 중 | 🔴 중지됨

**최근 작업:**
- 최근에 실행된 명령이나 작업의 이력이 표시됩니다

### 3.2 🐳 컨테이너

서버에서 실행 중인 **모든 Docker 컨테이너**를 테이블 형태로 보여줍니다.

| 열 | 설명 |
|----|------|
| **상태** | 컨테이너 실행 상태 (running / exited / created) |
| **이름** | 컨테이너 이름 |
| **이미지** | 사용 중인 Docker 이미지 |
| **상태 정보** | 가동 시간 (예: "Up 2 hours") |
| **포트** | 매핑된 포트 번호 |

> Orbitron으로 배포된 컨테이너 외에도 시스템 컨테이너(PostgreSQL, pgAdmin 등)도 함께 표시됩니다.

### 3.3 📦 프로젝트

Orbitron에 등록된 **웹 프로젝트 목록**입니다.

각 프로젝트 카드에는:
- 프로젝트 이름 + 실행 상태
- 서브도메인 주소
- 외부 접속 URL (있는 경우)

**사용 가능한 기능:**

| 버튼 | 기능 | 설명 |
|------|------|------|
| 🚀 **배포** | 프로젝트 재배포 | 클릭 시 확인 후 배포 시작되며, 알림이 표시됩니다 |
| 📋 **로그** | 컨테이너 로그 조회 | 최근 로그 50줄을 팝업으로 보여줍니다 |

> **주의**: 배포 버튼을 클릭하면 해당 프로젝트가 즉시 재빌드됩니다. 실행 중인 사이트에 영향을 줄 수 있으니 신중하게 사용하세요.

### 3.4 📋 작업 이력

지금까지 실행된 모든 명령과 작업의 기록을 시간순으로 보여줍니다.

| 아이콘 | 의미 |
|--------|------|
| ✅ | 완료된 작업 |
| ❌ | 실패한 작업 |
| 🔄 | 현재 실행 중인 작업 |
| ⏳ | 대기 중인 작업 |

각 항목에는 **실행한 명령**, **실행 시간**, **상태**, **작업 번호**가 표시됩니다.

### 3.5 💬 텔레그램 연동

텔레그램 봇을 통해 **모바일에서 서버를 관리**할 수 있습니다.

**설정 방법:**
1. Telegram에서 [@BotFather](https://t.me/BotFather)에게 `/newbot` 명령으로 봇 생성
2. 받은 토큰을 관리자에게 전달하여 서버에 등록
3. [@userinfobot](https://t.me/userinfobot)에서 자신의 Telegram ID 확인
4. 관리자에게 ID 전달하여 인증 등록

**사용 가능한 텔레그램 명령어:**

| 명령 | 설명 | 예시 |
|------|------|------|
| `/start` | 봇 사용 시작 & 인사 | `/start` |
| `/help` | 전체 명령어 도움말 | `/help` |
| `/status` | 시스템 전체 상태 리포트 | `/status` |
| `/containers` | Docker 컨테이너 목록 & 상태 | `/containers` |
| `/projects` | Orbitron 프로젝트 목록 | `/projects` |
| `/disk` | 디스크 사용량 상세 | `/disk` |
| `/uptime` | 서버 가동 시간 | `/uptime` |
| `/plan` | 구축계획서 요약 | `/plan` |
| `/logs [이름]` | 컨테이너 로그 조회 (최근 30줄) | `/logs wra` |
| `/deploy [이름]` | 프로젝트 원격 배포 실행 | `/deploy iiff` |

> `/logs`와 `/deploy` 명령은 뒤에 프로젝트 이름을 입력해야 합니다.

## 6. 문제 해결 (FAQ)

### Q. 로그인이 되지 않아요

**확인 사항:**
- 이메일 주소가 정확한지 확인하세요
- 비밀번호를 정확히 입력했는지 확인하세요 (6자 이상)
- 계정이 비활성화되었을 수 있습니다 → 관리자에게 문의

### Q. 대시보드에 데이터가 안 나와요

**확인 사항:**
- 서버가 정상 실행 중인지 확인하세요
- 브라우저를 새로고침(F5)해보세요
- 다른 브라우저에서 시도해보세요

### Q. 프로젝트 배포가 실패했어요

**확인 사항:**
- Orbitron 서버가 정상 작동 중인지 확인하세요
- 프로젝트 이름이 정확한지 확인하세요
- 로그 버튼으로 오류 내용을 확인하세요
- 반복해서 실패하면 관리자에게 문의하세요

### Q. 텔레그램 봇이 응답하지 않아요

**확인 사항:**
- 봇에게 먼저 `/start` 명령을 보내보세요
- 봇 토큰과 관리자 ID가 설정되었는지 텔레그램 연동 페이지에서 확인하세요
- 서버가 재시작된 경우 다시 `/start`를 보내세요

### Q. 내 작업 이력이 보이지 않아요

- 일반 사용자는 **본인이 실행한 작업만** 표시됩니다
- 아직 실행한 작업이 없으면 빈 목록이 정상입니다

## 7. 보안 안내

### 7.1 비밀번호 관리
- 비밀번호는 **bcrypt**로 암호화되어 저장됩니다
- 다른 사이트와 동일한 비밀번호를 사용하지 마세요
- 주기적으로 비밀번호를 변경하는 것을 권장합니다

### 7.2 로그인 세션
- 로그인 상태는 **7일간** 유지됩니다
- 7일이 지나면 자동으로 로그아웃되며 다시 로그인해야 합니다
- 공용 PC에서는 반드시 **로그아웃**하세요

### 7.3 감사 기록
- 로그인, 배포, 시스템 변경 등 모든 **중요 행동은 기록**됩니다
- 이 기록은 보안 관리 목적으로 관리자가 확인할 수 있습니다
