🚀 Latest Updates

Check all the newly added or improved features and changes in Orbitron at a glance.

v1.7.1 March 24, 2026

🐳 Docker Dependency Auto-Injection & Local DB Port Forwarding Support

  • Feature
    Next.js System OS Dependency Auto-Injection (apk add)

    Analyzes package.json and automatically injects required OS system packages (ffmpeg, python3, cairo, etc.) into the Alpine Linux-based Docker image at build time if media-related packages (yt-dlp, ffmpeg) or image processing libraries (canvas) are present. Additionally supports loading custom packages via apk.txt and apt.txt files located in the project root.

  • Improvement
    Database External Port Auto-Mapping

    When deploying managed databases such as PostgreSQL or Redis, the assigned port is now automatically port-forwarded (-p) and connected to the internal Docker port (5432, 6379). This maintains internal network isolation while allowing developers to connect directly from the outside during local development via DB tools like Prisma Studio or DBeaver.

v1.7.0 March 6, 2026

🌐 Auto Custom Domain Connection & Domain Management Roadmap

  • Feature
    Custom Domain Auto-Connection System

    Automatically connects domains purchased from Gabia, Namecheap, etc. in 3 steps (DNS Verification → Cloudflare Tunnel Connection → Activation). By just setting the CNAME record, SSL certificates, reverse proxy, and CDN are all automatically applied.

  • Feature
    Automated DNS Verification API

    Automatically verifies in real-time whether DNS records are set correctly before connecting a custom domain. Provides specific modification guides for incorrect settings.

  • Docs
    Domain Management Guide & Purchase Feature Roadmap

    Added CNAME setup guides for major domain registrars like Gabia, Namecheap, and Cloudflare. Revealed the implementation roadmap for future domain search/purchase/auto-connection features (Gabia Reseller API, Namecheap API).

v1.6.0 March 3, 2026

📱 KakaoTalk and Telegram Mobile Simulator & Unified Cross Authentication

  • Feature
    SSE-Equipped Telegram and KakaoTalk Mobile Web Simulator

    Launched a web simulator that perfectly mimics the mobile Telegram and KakaoTalk UI structures within the RemoteAGT dashboard so you can demonstrate perfect frontend control in the browser without an actual bot token. Delivers a seamless response experience through a real-time SSE (Server-Sent Events) asynchronous pipeline without polling and Webhook mocking technology.

  • Feature
    KakaoTalk Channel Auto Notification Bot Integration

    Now, deployment progress and server failure monitoring warnings can be received in real-time in mobile chat rooms through 'KakaoTalk Open Builder Skill Integration' tailored for the domestic environment, in addition to Telegram.

  • Improvement
    Orbitron & RemoteAGT Crossover Unified Authentication (SSO)

    Now, two completely independent services have been integrated over a single physical PostgreSQL database network. Whichever side you sign up from, fully cross-compatible login is possible through the *.twinverse.org wildcard cookie.

  • Bug Fix
    Database Schema Column Missing Resolved

    Completely fixed and migrated the critical login block bug that occurred due to the absence of web_user_id and user_id columns during the activity log (ragt_audit_log) logging process upon sign-up.

v1.5.0 March 2, 2026

🧠 Genius Deployer — Smart Project Analysis & Multi-Service Deployment

  • Feature
    Smart Project Analyzer — Automated Project Structure Analyzer

    100% auto-detects runtimes (Node.js, Python, Go, etc.), frameworks (over 20 types including FastAPI, Vite, Next.js), service types, ports, build commands, and dependencies upon deployment. Perfect deployment is possible even without orbitron.yaml.

  • Feature
    Multi-Service orbitron.yaml Support

    By defining multiple services such as backend, frontend, and staff apps as a services array in a single repository, it automatically deploys each service to the correct target (Docker/CF Pages). Automatically connects URLs between services using the from: reference syntax.

  • Feature
    Cloudflare Pages Static Site Auto-Deployment

    When a type: static service is detected, it automatically builds/deploys to the Cloudflare Pages CDN. Served at ultra-high speed from the global edge network, and also supports simultaneous multi-frontend deployments.

  • Feature
    PostDeploy Hooks — Execute Custom Commands Post-Deployment

    Defines the postDeploy: array in orbitron.yaml to automatically execute additional commands after the main deployment is complete. Supports environment variable injection, a 3-minute timeout, and success/failure logging.

  • Bug Fix
    Fixed Missing Environment Variables During Static Service Build

    Fixed a critical bug where environment variables such as VITE_API_URL were not injected during Vite/React static service builds, causing the production environment to request APIs to localhost.

  • Docs
    Massive Update of 5 Deployment Documentation Pages

    Added contents on multi-service deployment, Smart Analyzer, CF Pages auto deployment, and Error Knowledge DB to the IaC guide, static sites, AI analyzer, core concepts, and web services docs.

  • Feature
    Viewer Tier — Read-Only Member Tier Added

    New users are automatically assigned the 'Viewer' tier, allowing them to browse the dashboard menu but restricting project creation, deployment, and modification. Administrators can promote their tier (Viewer → User → Admin).
