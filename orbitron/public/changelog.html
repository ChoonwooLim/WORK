v2.3 2026년 4월 14일 커스텀 도메인 (멀티테넌트) Let's Encrypt 자동 SSL twinverse.org 마이그레이션
🌐 멀티테넌트 커스텀 도메인 — BYOD (Bring Your Own Domain)
  • Orbitron 유저가 소유한 도메인(예: myapp.com)을 어떤 프로젝트에도 자유롭게 연결 가능. A 레코드(apex 지원) 또는 CNAME 양방 지원
  • 대시보드의 프로젝트 설정 → 🌐 커스텀 도메인: DNS 가이드 자동 표시 (공인 IP + 터널 호스트네임 힌트), DNS 검증 버튼, 연결 버튼, 인증서 만료 카운트다운, 갱신 버튼, 연결 해제
  • 이전 버전의 tunnelService.addCustomDomainRoute() 의존성 제거 — 그 경로는 "우리 Cloudflare 계정의 zone 내 도메인"만 지원. 유저 소유 외부 도메인은 작동 불가였음. 완전히 재설계
  • 신규 DB 흐름: custom_domain 컬럼 + services/nginx.js의 server_name 자동 포함 + HTTPS 블록 자동 생성
🔐 Let's Encrypt HTTP-01 자동 SSL
  • 신규 services/letsencrypt.js — certbot webroot 모드 래퍼. issueCert / renewAll / revokeCert / certInfo 노출. sudo 없이 stevenlim 계정으로 실행되도록 --config-dir /home/stevenlim/letsencrypt/config 분리
  • 신규 services/nginx.js _httpsBlock() — 연결된 커스텀 도메인의 HTTPS 서버 블록을 자동 생성. TLSv1.2/1.3, MozSSL cipher 프로파일, ACME challenge 패스스루 포함
  • 신규 infrastructure/nginx/conf.d/default.conf에 shared ACME challenge 위치 — 모든 도메인이 /.well-known/acme-challenge/*를 webroot로 서빙
  • infrastructure/docker-compose.yml에 두 볼륨 추가 (read-only): /home/stevenlim/letsencrypt/config → /etc/letsencrypt, webroot-acme → /var/www/certbot. dev-nginx가 인증서 파일을 직접 읽고 challenge 파일을 서빙
  • 인증서 자동 갱신: ~/.config/systemd/user/orbitron-cert-renew.{service,timer} — 매일 00:45 + 0-1h 랜덤 지연. 갱신 후 자동 nginx reload
🎨 개선된 도메인 연결 UI
  • A 레코드 / CNAME 두 가지 옵션을 시각적으로 분리 표시 + 공인 IP 자동 노출 (/api/projects/:id/domain/status로 동적 로드)
  • 연결된 도메인에 인증서 남은 일수 실시간 표시 (30일 미만 시 경고색)
  • 🔄 갱신 버튼 — 별도 API (POST /api/projects/:id/domain/renew)로 즉시 재발급
  • DNS 전파 가이드 예시 (가비아, Cloudflare, Namecheap) + apex 도메인 주의사항

🌱 로컬 Gemma 4 E4B 통합 (Ollama 기반, API 키 불필요)
  • Google DeepMind Gemma 4 E4B (Q4_K_M, 8B 파라미터, 9.6GB) 모델을 로컬 Ollama daemon으로 서빙. 외부 API 비용 0원, 인터넷 연결 불필요, 데이터가 서버 밖으로 나가지 않음

  • Ollama daemon은 ~/.config/systemd/user/ollama.service 사용자 systemd 유닛으로 영속화 — 사용자 로그인 시 자동 시작 + 실패 시 자동 재시작

  • 3개 서비스 경로 모두 통합: aiAnalyzer.analyzeError() (로그 분석), aiAnalyzer.chat() (AI 채팅), aiAutoRepair.analyzeAndGeneratePatch() (자동 코드 패치 생성, JSON 모드)
  • 모델 ID 매핑 테이블 등록 (gemma-4-e2b/e4b/26b/31b → gemma4:e2b/e4b/26b/31b) — 추후 더 큰 variant pull 시 옵션만 추가하면 동작
  • OLLAMA_HOST 환경변수로 원격 Ollama 인스턴스 지원 (예: 다른 GPU 머신)
🛠️ 컨테이너 복구 로직 개선 — 고스트 재시작 방지 (포트 충돌 해결)
  • Blue-Green 배포 컨테이너는 orbitron-{subdomain}-{deployHash} 형식인데, 서버 시작 시 복구 로직이 정확 이름 orbitron-{subdomain}로만 매칭하여 살아있는 컨테이너를 못 찾고 새로 만들려다 호스트 포트 충돌(Bind for 0.0.0.0:3480 failed) 발생하던 버그 수정
  • 이제 docker ps --filter "name=^orbitron-{sub}" 으로 prefix 매칭하여 살아있는 컨테이너를 우선 재사용. 정확 일치 우선, 없으면 가장 최근 deployHash 컨테이너 채택
  • 살아있는 컨테이너 발견 시 --restart unless-stopped 정책만 갱신하고 container_id 컬럼을 동기화 — 불필요한 컨테이너 재생성 0건
🛠️ spawn E2BIG 진짜 원인 발견 및 해결
  • 서버 시작 복구 경로가 project.env_vars를 복호화하지 않고 dockerService.startContainer()에 그대로 전달하던 결정적 버그
  • 그 결과 Object.keys(envVars)가 암호화된 hex 문자열을 string으로 받아 각 문자를 키로 인식 → -e 0='2' -e 1='c' -e 2='3' ... × 12,748글자 → 단일 명령어 길이가 ~128KB로 MAX_ARG_STRLEN 한계에 도달 → execve가 E2BIG 반환
  • 1차 수정: server.js에 decryptProjectEnvVars() 헬퍼 추가하여 복구 루프에서 startContainer 호출 직전 복호화 보장
  • 2차 수정 (defense in depth): docker.js startContainer()에 type guard 추가 — env_vars가 plain object가 아니면 빈 객체로 강제하고 경고 로그 출력
🛠️ 손상된 env_vars 행 복구 — Decryption failing 로그 스팸 제거
  • 이전 JWT_SECRET으로 암호화된 채로 남아있던 4개 프로젝트 행(Twinverse, RemoteAGT, KcontentsHub, K-ContentHub_DB)이 매 요청마다 Decryption failing (wrong key or corrupted data) 로그를 발생시켜 error.log가 31MB까지 팽창
  • 안전한 복구 절차 수행: 손상된 행의 raw 값을 backups/broken-env-vars-2026-04-11.json에 메타데이터와 함께 백업 → DB 행을 빈 객체 {}로 reset (현재 키로 재암호화) → 이전 키를 나중에 발견하면 복구 가능
  • 최종 reload 후 30초 모니터링 결과 Decryption failing / spawn E2BIG / Bind for 0.0.0.0 에러 0건 확인

    터널 지수 백오프 & Graceful Shutdown

    Cloudflare 터널 재연결 시 5초→10초→20초→최대 5분 간격의 지수 백오프가 적용됩니다. 서버 종료 시 DB 커넥션 풀을 안전하게 정리하는 Graceful Shutdown이 추가되었습니다.

  • 개선
    비동기 I/O 전환 & exec 버퍼 최적화

    실제 봇 토큰 없이도 브라우저 내에서 완벽한 프론트엔드 제어를 시연할 수 있도록 RemoteAGT 대시보드 내에 모바일 텔레그램 및 카카오톡 UI 구조를 완벽하게 모사한 웹 시뮬레이터를 런칭했습니다. 폴링 없는 실시간 SSE(Server-Sent Events) 비동기 파이프라인과 Webhook 모킹 기술을 통해 매끄러운 응답 경험을 선사합니다.

  • 신기능
    카카오톡 채널 자동 알림봇 연동

    이제 배포 진행 상황, 서버 장애 모니터링 경고 등을 텔레그램뿐만 아니라 국내 환경에 맞춘 '카카오톡 오픈빌더 스킬 연동'을 통하여 모바일 톡방에서 실시간으로 수신할 수 있습니다.

  • 개선
    Orbitron & RemoteAGT 크로스오버 통합 인증(SSO)

    이제 완전히 독립된 두 서비스가 물리적인 단일 PostgreSQL 데이터베이스 통신망 위에서 통합되었습니다. 어느 쪽에서 가입하든 *.twinverse.org 와일드카드 쿠키를 통해 완벽한 크로스 호환 로그인이 가능합니다.

  • 버그수정
    데이터베이스 Schema Column 누락 해결

    가입 시 활동 로그(ragt_audit_log) 로깅 과정에서 web_user_id 및 user_id 컬럼 부재로 발생하던 치명적 로그인 블록 버그를 완벽히 수정 및 마이그레이션 조치했습니다.

v1.5.0 2026년 3월 2일

🧠 Genius Deployer — 스마트 프로젝트 분석 & 멀티 서비스 배포

  • 신기능
    Smart Project Analyzer — 프로젝트 구조 자동 분석기

    배포 시 런타임(Node.js, Python, Go 등), 프레임워크(FastAPI, Vite, Next.js 등 20종+), 서비스 타입, 포트, 빌드 명령어, 의존관계를 100% 자동 감지합니다. orbitron.yaml 없이도 완벽한 배포가 가능합니다.

  • 신기능
    멀티 서비스 orbitron.yaml 지원

    하나의 레포에 백엔드, 프론트엔드, 직원앱 등 여러 서비스를 services 배열로 정의하면 각 서비스를 자동으로 올바른 타겟(Docker/CF Pages)에 배포합니다. from: 참조 문법으로 서비스 간 URL을 자동 연결합니다.

  • 신기능
    Cloudflare Pages 정적 사이트 자동 배포

    type: static 서비스를 감지하면 Cloudflare Pages CDN에 자동 빌드/배포합니다. 전 세계 에지 네트워크에서 초고속으로 서빙되며, 멀티 프론트엔드 동시 배포도 지원합니다.

  • 신기능
    PostDeploy 훅 — 배포 후 커스텀 명령 실행

    orbitron.yaml에 postDeploy: 배열을 정의하여 메인 배포 완료 후 추가 명령어를 자동 실행합니다. 환경변수 주입, 3분 타임아웃, 성공/실패 로그 기록을 지원합니다.

  • 버그수정
    정적 서비스 빌드 시 환경변수 미주입 수정

    Vite/React 정적 서비스 빌드 시 VITE_API_URL 등 환경변수가 주입되지 않아 프로덕션에서 localhost로 API를 요청하던 크리티컬 버그를 수정했습니다.

  • 문서
    배포 문서 5개 페이지 대규모 업데이트

    IaC 가이드, 정적 사이트, AI 분석기, 핵심 개념, 웹 서비스 문서에 멀티 서비스 배포, Smart Analyzer, CF Pages 자동 배포, Error Knowledge DB 내용을 추가했습니다.

  • 신기능
    Viewer 등급 — 읽기 전용 회원 등급 추가

    신규 가입자는 자동으로 'Viewer' 등급이 부여되어, 대시보드 메뉴를 둘러볼 수 있지만 프로젝트 생성·배포·수정은 불가합니다. 관리자가 등급을 승격(Viewer → User → Admin)할 수 있습니다.
