RemoteAGT v1.2.0 2026년 4월 3일

🏗️ RemoteAGT 대규모 아키텍처 리팩토링

  • 리팩토링
    server.js 748줄 → 230줄 (-69%) 경량화

    텔레그램·카카오 시뮬레이터 라우트(~370줄, 80% 중복 코드)를 src/routes/simulator.js로 분리하고, 공통 명령 핸들러 handleSimulatorCommand() 하나로 통합했습니다. SSE 클라이언트 관리도 createSSEHandler() 팩토리 함수로 일원화되었습니다.

  • 보안
    마크다운 렌더러 XSS 완전 차단 (marked + DOMPurify)

    15개 정규식 체인으로 구현된 자체 마크다운 렌더러를 marked + DOMPurify로 교체했습니다. [click](javascript:alert(1)), <script>, <img onerror=...> 등 3가지 XSS 공격 벡터가 모두 차단됩니다.

  • 개선
    레거시 API 경로 호환성 유지

    기존 /api/telegram/simulate, /api/kakao/simulate 등의 엔드포인트가 새로운 /api/simulator/* 경로로 자동 프록시되어 프론트엔드 변경 없이 원활하게 동작합니다.

v1.8.1 2026년 4월 3일

🔬 코드 리뷰 기반 치명적 버그 긴급 수정

  • 긴급 수정
    Cloudflare Named 터널 생성 실패 원인 수정

    _writeConfig 메서드가 async로 전환된 후 await 없이 호출되어 configPath가 [object Promise] 문자열로 전달되던 치명적 버그가 수정되었습니다. 이 버그로 인해 모든 Named 터널이 시작에 실패했습니다.

  • 긴급 수정
    중복 stopTunnel 메서드 병합 (Systemd 터널 미정지 해결)

    stopTunnel()이 2번 정의되어 Systemd 터널 정지 코드가 Dead Code였던 문제가 수정되었습니다. 이제 Systemd·Quick·Legacy 터널이 모두 올바르게 정지됩니다.

  • 버그 수정
    환경변수 암호화 따옴표 버그 & Health Check 포트 불일치

    orbitron.yaml env 오버라이드 시 암호화 값에 여분의 따옴표(")가 감싸져 복호화 실패하던 버그, Health Check가 자동 변경된 포트 대신 원래 포트를 사용하던 버그, 배포 성공 후 60분 타이머 미해제 메모리 누수, apk.txt 개행 파싱 regex 오류가 모두 수정되었습니다.

v1.8.0 2026년 4월 3일

🛡️ 엔터프라이즈급 안정성 & 보안 최적화

  • 신기능
    Docker 레이어 캐싱 기본 활성화

    매 배포마다 전체 리빌드를 수행하던 --no-cache 옵션이 제거되고, Docker 빌드 레이어 캐싱이 기본 활성화되었습니다. package.json이 변경되지 않은 경우 npm install 단계를 건너뛰어 빌드 속도가 2~3배 향상됩니다. 필요 시 환경변수 DOCKER_NO_CACHE=true로 캐시를 비활성화할 수 있습니다.

  • 신기능
    60분 배포 타임아웃 보호

    모든 배포에 자동 타임아웃이 적용됩니다. 네트워크 장애나 무한 루프 빌드로 프로세스가 멈추더라도 60분 경과 시 자동 중단되어 서버 리소스 고갈을 원천 차단합니다.

  • 신기능
    터널 도메인 환경변수 커스터마이징

    기본 도메인(twinverse.org)을 TUNNEL_DOMAIN 환경변수로 변경할 수 있게 되었습니다. 자체 Cloudflare 도메인 보유 시 모든 프로젝트의 서브도메인을 커스텀 도메인으로 배포 가능합니다.

  • 개선
    JWT 시크릿 보안 강화

    하드코딩된 JWT 폴백 시크릿('orbitron-secret-key')이 완전히 제거되었습니다. JWT_SECRET 환경변수가 필수로 요구되며, 미설정 시 서버가 시작되지 않습니다.

  • 개선
    DB 커넥션 풀 제한 및 에러 핸들링

    PostgreSQL 커넥션 풀에 최대 20개 연결, 30초 유휴 타임아웃, 5초 연결 타임아웃이 적용되었습니다. 예기치 않은 풀 에러도 자동 로깅됩니다.

  • 개선
    배포 로그 512KB 제한 & Docker 이미지 자동 정리

    배포 로그가 512KB를 초과하면 자동 잘림 처리되어 DB 팽창을 방지합니다. 또한 배포 완료 후 24시간 이상 미사용된 Docker 이미지를 자동 삭제하여 디스크 공간을 확보합니다.

  • 개선
    터널 지수 백오프 & Graceful Shutdown

    Cloudflare 터널 재연결 시 5초→10초→20초→최대 5분 간격의 지수 백오프가 적용됩니다. 서버 종료 시 DB 커넥션 풀을 안전하게 정리하는 Graceful Shutdown이 추가되었습니다.

  • 개선
    비동기 I/O 전환 & exec 버퍼 최적화

    Nginx 설정 및 터널 설정 파일 쓰기가 비동기(async)로 전환되어 이벤트 루프 블로킹이 해소되었습니다. exec() 호출의 maxBuffer도 50MB에서 10MB로 최적화되어 메모리 효율이 개선됩니다.

v1.7.1 2026년 3월 24일

🐳 Docker 의존성 자동 주입 & 로컬 DB 포트포워딩 지원

  • 신기능
    Next.js 시스템 OS 의존성 자동 주입 (apk add)

    package.json을 분석하여 미디어 패키지(yt-dlp, ffmpeg)나 이미지 처리(canvas) 라이브러리가 감지될 경우, 빌드 시 필요한 OS 단위의 시스템 패키지(ffmpeg, python3, cairo 등)를 Alpine 리눅스 Docker 이미지에 자동 내장(Inject)합니다. 최상단 디렉터리의 apk.txt 및 apt.txt 파일 로드 기능도 추가 지원합니다.

  • 개선
    데이터베이스 외부 포트 자동 매핑

    PostgreSQL, Redis 등 매니지드 데이터베이스 배포 시 지정된 동적 포트(예: 3776)를 도커 내부 포트(5432, 6379)와 자동으로 포트포워딩(-p) 연결합니다. 이를 통해 내부 격리를 유지하면서도 로컬 DBeaver, Prisma Studio 등을 외부에서 곧바로 연동할 수 있게 개선되었습니다.

    이제 배포 진행 상황, 서버 장애 모니터링 경고 등을 텔레그램뿐만 아니라 국내 환경에 맞춘 '카카오톡 오픈빌더 스킬 연동'을 통하여 모바일 톡방에서 실시간으로 수신할 수 있습니다.

  • 개선
    Orbitron & RemoteAGT 크로스오버 통합 인증(SSO)

    이제 완전히 독립된 두 서비스가 물리적인 단일 PostgreSQL 데이터베이스 통신망 위에서 통합되었습니다. 어느 쪽에서 가입하든 *.twinverse.org 와일드카드 쿠키를 통해 완벽한 크로스 호환 로그인이 가능합니다.

  • 버그수정
    데이터베이스 Schema Column 누락 해결

    가입 시 활동 로그(ragt_audit_log) 로깅 과정에서 web_user_id 및 user_id 컬럼 부재로 발생하던 치명적 로그인 블록 버그를 완벽히 수정 및 마이그레이션 조치했습니다.

v1.5.0 2026년 3월 2일

🧠 Genius Deployer — 스마트 프로젝트 분석 & 멀티 서비스 배포

  • 신기능
    Smart Project Analyzer — 프로젝트 구조 자동 분석기

    배포 시 런타임(Node.js, Python, Go 등), 프레임워크(FastAPI, Vite, Next.js 등 20종+), 서비스 타입, 포트, 빌드 명령어, 의존관계를 100% 자동 감지합니다. orbitron.yaml 없이도 완벽한 배포가 가능합니다.

  • 신기능
    멀티 서비스 orbitron.yaml 지원

    하나의 레포에 백엔드, 프론트엔드, 직원앱 등 여러 서비스를 services 배열로 정의하면 각 서비스를 자동으로 올바른 타겟(Docker/CF Pages)에 배포합니다. from: 참조 문법으로 서비스 간 URL을 자동 연결합니다.

  • 신기능
    Cloudflare Pages 정적 사이트 자동 배포

    type: static 서비스를 감지하면 Cloudflare Pages CDN에 자동 빌드/배포합니다. 전 세계 에지 네트워크에서 초고속으로 서빙되며, 멀티 프론트엔드 동시 배포도 지원합니다.

  • 신기능
    PostDeploy 훅 — 배포 후 커스텀 명령 실행

    orbitron.yaml에 postDeploy: 배열을 정의하여 메인 배포 완료 후 추가 명령어를 자동 실행합니다. 환경변수 주입, 3분 타임아웃, 성공/실패 로그 기록을 지원합니다.

  • 버그수정
    정적 서비스 빌드 시 환경변수 미주입 수정

    Vite/React 정적 서비스 빌드 시 VITE_API_URL 등 환경변수가 주입되지 않아 프로덕션에서 localhost로 API를 요청하던 크리티컬 버그를 수정했습니다.

  • 문서
    배포 문서 5개 페이지 대규모 업데이트

    IaC 가이드, 정적 사이트, AI 분석기, 핵심 개념, 웹 서비스 문서에 멀티 서비스 배포, Smart Analyzer, CF Pages 자동 배포, Error Knowledge DB 내용을 추가했습니다.

  • 신기능
    Viewer 등급 — 읽기 전용 회원 등급 추가

    신규 가입자는 자동으로 'Viewer' 등급이 부여되어, 대시보드 메뉴를 둘러볼 수 있지만 프로젝트 생성·배포·수정은 불가합니다. 관리자가 등급을 승격(Viewer → User → Admin)할 수 있습니다.
