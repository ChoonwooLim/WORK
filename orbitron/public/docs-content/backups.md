# 하이브리드 미디어 및 시스템 백업 (Backups)

프로젝트 운영 중 유저가 업로드한 소중한 프로필 사진들(`uploads/` 폴더 안의 이미지 등)이나, 핵심 코딩 파일이 실수로 지워지는 대참사를 방지하는 강력한 데이터 보존 스냅샷 기능입니다.

PostgreSQL 같은 관계형 데이터베이스 백업 외에도, 물리적인 '파일' 자체를 안전하게 보관해야 할 때 사용합니다. Orbitron은 두 가지 완벽한 하이브리드 방식의 파일 아카이빙 솔루션을 대시보드 안에서 버튼 하나로 제공합니다.

---

## 1. 미디어 시스템 백업 (Media Volumn Backup)

Docker 컨테이너 안은 기본적으로 코드를 껐다 킬 때마다 데이터가 휘발되는(Stateless) 모래성 같은 환경입니다. 만약 유저가 게시판에 올린 사진(`media/` 폴더)들을 서버를 끌 때마다 날려버리고 싶지 않다면, Orbitron 생성 시 영구 저장소 볼륨 설정(`Volumes`)을 거쳤을 것입니다.

이 소중한 마운트 폴더 자체를 물리적으로 시스템 깊숙한 곳의 안전한 방공호로 통째로 압축해서 보관하는 기능입니다.

### 백업 / 복구 방법
1. 대시보드 프로젝트 상세 페이지로 진입하여 우측의 **데이터 관리 (Data Management)** 패널을 확인합니다.
2. **`미디어 시스템 백업`** 주황색 버튼: 클릭 즉시 현재 프로젝트 내부 미디어 볼륨 폴더가 시스템의 안전한 로컬 방공호 경로를 향해 통째로 `.tar` 아카이빙 압축되어 스냅샷을 찍습니다.
3. 이 스냅샷은 프로젝트가 완전히 삭제되더라도 디스크에 남아 언제든 복구가 가능합니다.

## 2. Git 프로젝트 클론 복제 백업 

수일 동안 작성한 깃허브 오픈소스 저장소가 누군가에 의해 해킹되어 통째로 날아가거나 비공개로 바뀌어버렸다면 어떡할까요? 서버는 살아있는데 원본 소스 코드가 사라지는 끔찍한 상황입니다.

Orbitron은 현재 연동된 원격 소스 코드 저장소(GitHub Repo URL) 자체를 대형 메인프레임 안에 은밀하고 깊숙하게 별도의 `git clone` 및 주기적 `git pull` 방식으로 최신화하여 백업해내는 백그라운드 엔진을 갖추고 있습니다. 

### 작동 방식
1. 프로젝트 우측 화면의 **`Git 프로젝트 클론 복제 백업`** 버튼을 클릭합니다.
2. Orbitron의 내부 데몬 프로세스가 여러분이 처음 연동했던 원본 코드를 가져와 시스템의 숨겨진 디렉토리(`/home/stevenlim/GitClones/` 등)에 완벽한 사본(Clone)을 생성합니다.
3. 만약 이미 과거에 백업해 둔 폴더가 있다면, 전체를 다시 다운받는 멍청한 짓을 하지 않고 오직 변경된 최신 로그만 영리하게 `pull` 하여 병합(Merge)해 냅니다.
4. 배포 시스템과 완벽히 물리적으로 차단·격리된 이 사본 저장소는, 원본이 폭파되어도 그 자체로 여러분의 든든한 방어막이 되어 줍니다.
