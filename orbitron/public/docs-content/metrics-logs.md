# 실시간 모니터링 메트릭 및 로그 

> 💡 **출시 예정 기능 (Coming Soon)**
> 
> 이 기능은 현재 Orbitron 엔지니어링 팁에서 열심히 개발 중인 핵심 로드맵 기능입니다.
> 

현재 대시보드의 기초적인 상태 표시를 넘어, 서버의 폐 속에 들어간 공기량까지 꿰뚫어 보는 엑스레이(X-Ray) 모니터링 시스템입니다. 

*   **실시간 컴퓨팅 자원 그래프:** 1초 단위로 업데이트되는 미려한 그래프를 통해, 여러분의 컨테이너가 갉아먹는 CPU 사용량, Memory(RAM) 점유율, 외부 네트워크 I/O(수신/송신 트래픽) 대역폭을 한눈에 파악하세요.
*   **중앙 집중식 라이브 로그 (Tail Logs):** `docker logs -f`를 칠 필요 없이, 웹 기반 터미널에서 서버가 토해내는 모든 `console.log`와 `print()` 출력문을 실시간 스크롤로 감상할 수 있습니다.
*   **히스토리컬 데이터:** 과거 7일간의 메트릭 데이터를 보관하여 "도대체 어제 새벽 3시에 왜 서버가 재시작 되었는가?"를 사후 분석할 수 있습니다.
