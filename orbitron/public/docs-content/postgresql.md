# PostgreSQL 완전 관리형 데이터베이스 (Managed DB)

회원가입 정보, 게시글, 결제 내역 등 절대 지워지면 안 되는 영구적인 데이터(Persistent Data)를 저장할 깨끗한 빈 그릇이 필요하신가요? 

터미널 명령어를 입력할 필요 없이 대시보드에서 버튼 두 번이면, 즉시 접속 가능한 안정적인 원격 PostgreSQL 데이터베이스가 프로비저닝(할당) 됩니다.

---

## 🗄 발급 및 접속 (Connection String)

1. Orbitron 대시보드에서 `+ New Project` 버튼을 누른 후, 모달 창 상단의 **🗄 데이터베이스** 탭을 클릭합니다.
2. 엔진 종류에서 `PostgreSQL`을 고르고 생성합니다.
3. 몇 초 후, 프로젝트 카드에 가장 중요한 **Connection String (접속 주소)**이 나타납니다.

```
postgresql://orbit_user:secretpass123@your-db-host:5432/orbit_db
```

이 긴 주소 한 줄 안에는 유저명(`orbit_user`), 비밀번호(`secretpass123`), 주소, 그리고 실제 데이터베이스 이름(`orbit_db`)이 모두 합쳐져 있습니다. 이 주소를 통째로 복사합니다.

### 코드 연동 (환경변수 주입)

여러분이 배포 중인 웹 앱(Django, NestJS, Prisma 등)의 설정 파일이나 코드에 이 주소를 절대 하드코딩하지 마세요!
대신 해당 웹 앱 프로젝트 설정(Settings)에서 **환경 변수(Environment Variables)** 항목에 `DATABASE_URL` 이라는 이름으로 값을 등록해 두면, 서버가 켜질 때 프레임워크가 알아서 이 DB와 통신을 개시합니다.

---

## 🔒 프라이빗 보안 정책

발급된 데이터베이스는 기본적으로 완전히 **Lock(잠금)** 상태입니다.
구글 클라우드나 AWS처럼 인터넷에 5432 포트를 활짝 열어두지 않습니다. (해커의 브루트포스 공격 완벽 차단)

*   **내부망 접속:** Orbitron 시스템 안에서 구동되는 당신의 앱(웹 서비스 컨테이너)에서 접속(`postgresql://...`)을 시도할 때만 내부 가상 공유기(vRouter)가 이를 허락합니다.
*   **외부 도구 접속 불가:** 만약 여러분의 집에 있는 컴퓨터(로컬)에서 DBeaver나 DataGrip 같은 GUI 툴로 이 클라우드 DB에 직접 붙으려고 하면 연결이 거부(Timeout)됩니다. 외부에서 접속하려면 [프록시 터널링(진행 예정)](#/networking) 등의 우회 수단이 필요합니다.

## 데이터 보존 및 요금 

Orbitron에 배포한 소스 코드 컨테이너(웹 앱 서비스)는 언제든 삭제하고 다시 빌드해도 코드가 살아있지만, 데이터베이스는 삭제 버튼을 누르는 즉시 영구적으로 파기됩니다.
스토리지는 SSD 기반으로 제공되며, 과도한 용량 증설이 필요할 시 [스토리지 요금제 가이드](/pricing.html)를 참고해 주십시오.
