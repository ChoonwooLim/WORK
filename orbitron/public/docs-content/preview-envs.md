# 프리뷰 환경 (Preview Environments)

> 💡 **출시 예정 기능 (Coming Soon)**
> 
> 이 기능은 현재 Orbitron 엔지니어링 팁에서 열심히 개발 중인 핵심 로드맵 기능입니다.
> 

코드를 메인 브랜치(`main` 또는 `master`)에 병합하기 전, 팀원들이 직접 브라우저에서 버튼을 눌러보고 테스트할 수 있다면 얼마나 좋을까요? 

GitHub 저장소에서 새로운 **Pull Request (PR)** 를 열 때마다, Orbitron 시스템이 이를 즉각 감지하고 **해당 PR만을 위한 독립적인 임시 서버와 고유 접속 주소(URL)**를 1초 만에 띄워줍니다. 

*   QA 팀과 기획자는 코드를 직접 로컬에 클론받지 않아도 즉각적으로 변경된 UI를 눈으로 확인하고 피드백을 줄 수 있습니다.
*   PR이 메인 브랜치에 병합되거나(Merged) 닫히면(Closed), 할당되었던 프리뷰용 임시 컨테이너는 즉각 삭제되어 리소스를 낭비하지 않습니다.
