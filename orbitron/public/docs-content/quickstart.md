# 빠른 배포 가이드 (Quickstart)

Orbitron은 복잡한 서버 인원이나 쿠버네티스(Kubernetes) 같은 인프라 지식 없이도, 누구나 단 몇 번의 클릭으로 코드를 라이브 서비스로 만들 수 있게 해주는 **최고의 전자동 배포 대시보드**입니다.

이 가이드에서는 여러분의 첫 번째 애플리케이션을 인터넷에 공개하는 가장 빠르고 쉬운 방법을 안내합니다.

---

## 🚀 첫 프로젝트 배포하기

가장 기본이 되는 첫 프론트엔드 또는 백엔드 애플리케이션을 배포해 보겠습니다.

### 단계 1: 대시보드 접속
1. Orbitron 메인 홈페이지 상단 메뉴 바의 **[대시보드 로그인]** 버튼을 클릭하여 시스템에 접속합니다.
2. 계정이 없다면 깃허브(GitHub) 계정 연동을 통해 1초 만에 가입과 로그인을 동시에 진행할 수 있습니다.

### 단계 2: 새 프로젝트 생성
1. 대시보드 메인 화면(Overview) 우측 상단의 보라색 **`+ New Project`** 버튼을 누릅니다.
2. 펼쳐지는 요약 창에서 배포하고자 하는 서비스의 유형(Web Service, Static Site 등)을 선택합니다. 처음이시라면 가장 기본이 되는 **🌐 Web Service**를 추천합니다.

### 단계 3: 소스 코드 연결
가장 중요한 소스 코드 연동 단계입니다. Orbitron은 두 가지 방식을 모두 완벽하게 지원합니다.

*   **방식 A (추천): GitHub Repository 연동**
    *   화면에 표시된 텍스트 박스에 본인의 **GitHub 레포지토리 URL**을 복사해서 붙여넣습니다. (예: `https://github.com/username/my-awesome-app`)
    *   이 방식을 사용하면 코드가 수정될 때마다 자동으로 재배포가 되는 **CI/CD 기능**을 즉시 사용할 수 있습니다.
*   **방식 B: 간편 ZIP 업로드**
    *   아직 깃허브를 다루기 어렵거나, 단순 HTML 파일들만 배포하고 싶으신가요?
    *   소스 코드가 담긴 폴더를 통째로 `.zip` 파일로 압축한 뒤, 브라우저 화면 안으로 드래그 앤 드롭(끌어다 놓기) 하시면 끝입니다!

### 단계 4: 배포 시작
1. 소스 코드가 정상적으로 인식되면 하단의 **[프로젝트 생성]** 버튼이 활성화됩니다.
2. 과감하게 버튼을 누르세요! 

> 💡 **초보자도 안심하세요!**
> 도메인을 연결하거나, 포트포워딩을 하거나, Nginx를 직접 설정할 필요가 전혀 없습니다. NVM 설치, 패키지 다운로드, 방화벽 개방 등 배포에 필요한 수십 가지 귀찮은 터미널 작업들을 Orbitron 시스템이 1~2분 안에 알아서 뒷단에서 모두 처리합니다.

단 1~2분 뒤, 고유한 접속 보안 주소 (예: `https://my-awesome-app.twinverse.org`) 가 부여되며 여러분의 서비스가 전 세계에 공개됩니다! 🎉
