# 🪐 RemoteAGT — AI 원격 지휘·통제 시스템

RemoteAGT는 Orbitron과 연동되는 **차세대 원격 관제 시스템**입니다. 스마트폰의 텔레그램이나 카카오톡에서 AI(Antigravity)와 대화하며 코드를 수정하고, 서버 상태를 확인하고, 프로젝트를 배포할 수 있습니다.

> 💡 **한 줄 요약:** 카페에 앉아 스마트폰으로 "서버 상태 알려줘", "이 버그 고쳐줘", "배포해줘"라고 대화하면 AI가 다 해주는 시스템입니다.

---

## 🧠 핵심 기능: Antigravity AI 원격 개발

RemoteAGT의 가장 강력한 기능입니다. 텔레그램·카카오톡 메신저에서 Antigravity AI와 실시간 대화를 나누면서 개발 업무를 수행할 수 있습니다.

### 무엇이 가능한가요?

| 기능 | 예시 대화 |
|------|-----------|
| 🐛 버그 수정 | _"로그인 페이지에서 한글 입력이 안 돼, 고쳐줘"_ |
| ✨ 기능 추가 | _"상단 메뉴에 RemoteAGT 바로가기 넣어줘"_ |
| 📦 배포 | _"수정한 거 바로 배포해줘"_ |
| 🔍 코드 분석 | _"현재 서버의 인증 로직 구조 설명해줘"_ |
| 📊 상태 확인 | _"서버 CPU랑 메모리 얼마나 쓰고 있어?"_ |

### 왜 혁신적인가요?

기존 개발 방식은 반드시 **노트북을 열고 → IDE를 실행하고 → 코드를 수정하고 → 커밋하고 → 배포**해야 했습니다. RemoteAGT를 사용하면 이 모든 과정이 **스마트폰 채팅 한 번**으로 완료됩니다.

출퇴근 지하철에서, 점심 식사 중에, 심지어 침대에 누워서도 AI와 대화하며 개발할 수 있습니다.

## 📡 RemoteAGT 웹 대시보드

RemoteAGT는 전용 웹 대시보드(`https://RemoteAGT.twinverse.org`)를 제공합니다.

### 대시보드에서 볼 수 있는 것들

- **🖥 시스템 리소스**: CPU 사용률, 메모리 점유율, 디스크 용량을 시각적 프로그레스바로 실시간 확인
- **🐳 컨테이너 현황**: Orbitron에서 가동 중인 모든 Docker 컨테이너 목록, 상태, 이미지 정보
- **📦 프로젝트 목록**: 배포된 모든 Orbitron 프로젝트를 한눈에 파악
- **📋 작업 이력**: 누가 언제 어떤 배포/관리 작업을 수행했는지 기록

### 관리자 전용 기능

최상위 관리자(SuperAdmin)로 로그인하면 추가 메뉴가 나타납니다:

- **👥 사용자 관리**: 전체 사용자 목록 조회, 역할 변경(User/Admin/SuperAdmin), 계정 활성/비활성 제어
- **📜 활동 로그**: 로그인, 배포, 설정 변경 등 모든 사용자 활동이 시간순으로 기록된 감사 로그(Audit Log) 조회
- **📊 관리자 통계**: 전체 사용자 수, 주간 활성 유저, 오늘 로그인 수 등

## 💬 텔레그램 봇 연동

텔레그램 앱에서 RemoteAGT 봇과 대화하여 서버를 원격 관리할 수 있습니다.

### 사용 가능한 명령어

| 명령어 | 설명 | 사용 예시 |
|--------|------|-----------|
| `/start` | 봇 시작 및 환영 메시지 | `/start` |
| `/help` | 전체 명령어 도움말 | `/help` |
| `/status` | CPU, 메모리, 디스크, 컨테이너 수 등 시스템 전체 현황 리포트 | `/status` |
| `/containers` | 실행 중인 모든 Docker 컨테이너 목록 | `/containers` |
| `/projects` | Orbitron에 등록된 프로젝트 목록 | `/projects` |
| `/deploy <이름>` | 특정 프로젝트를 원격으로 재배포 | `/deploy my-website` |
| `/logs <이름>` | 컨테이너의 최근 로그 30줄 조회 | `/logs my-website` |
| `/disk` | 디스크 사용량 상세 정보 | `/disk` |
| `/plan` | RemoteAGT 구축계획서 보기 | `/plan` |
| `/uptime` | RemoteAGT 서버 가동 시간 확인 | `/uptime` |

### 설정 방법 (초보자용 단계별 안내)

텔레그램 봇을 처음 만들어보시는 분도 따라하실 수 있도록 쉽게 안내합니다.

**1단계: 텔레그램 봇 만들기**

1. 스마트폰에서 텔레그램 앱을 엽니다.
2. 검색창에 `@BotFather`를 입력하고 대화를 시작합니다.
3. `/newbot` 이라고 입력합니다.
4. 봇의 이름을 입력합니다. (예: `내서버관리봇`)
5. 봇의 사용자명을 입력합니다. (예: `my_server_bot`) — 반드시 `_bot`으로 끝나야 합니다.
6. 생성이 완료되면 **토큰(Token)** 이 표시됩니다. 이것을 복사해 두세요.

> 토큰 예시: `7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

**2단계: 내 텔레그램 ID 확인하기**

1. 텔레그램에서 `@userinfobot`을 검색하고 대화를 시작합니다.
2. 아무 메시지나 보내면 내 **숫자형 ID**가 표시됩니다. 이것도 복사해 두세요.

> ID 예시: `123456789`

**3단계: 서버에 설정 적용하기**

RemoteAGT의 `.env` 파일에 아래 두 값을 입력합니다:

```
TELEGRAM_BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_ADMIN_ID=123456789
```

**4단계: 서버 재시작**

```bash
pm2 restart remoteagt
```

이제 텔레그램에서 만들었던 봇에게 `/start`를 보내면 RemoteAGT가 응답합니다! 🎉

## 💛 카카오톡 챗봇 연동

한국에서 가장 많이 사용하는 메신저, 카카오톡에서도 동일한 원격 관제가 가능합니다.

### 사용 가능한 명령어

텔레그램과 동일한 명령어를 카카오톡에서도 사용할 수 있습니다:

| 명령어 | 설명 |
|--------|------|
| `/status` | 시스템 전체 현황 리포트 |
| `/containers` | 컨테이너 목록 |
| `/projects` | 프로젝트 목록 |
| `/deploy <이름>` | 프로젝트 배포 |
| `/logs <이름>` | 컨테이너 로그 조회 |
| `/disk` | 디스크 사용량 |
| `/help` | 도움말 |

### 설정 방법 (초보자용 단계별 안내)

카카오톡 챗봇 연동은 텔레그램보다 단계가 조금 더 있지만, 천천히 따라하시면 충분합니다.

**1단계: 카카오 개발자 앱 만들기**

1. [카카오 Developers](https://developers.kakao.com)에 접속하여 로그인합니다.
2. **내 애플리케이션 → 애플리케이션 추가하기**를 클릭합니다.
3. 앱 이름(예: `RemoteAGT`)을 입력하고 저장합니다.
4. 생성된 앱의 **REST API 키**를 복사해 둡니다.

**2단계: 카카오톡 채널 만들기**

1. [카카오톡 채널](https://center-pf.kakao.com)에 접속합니다.
2. **새 채널 만들기**를 클릭하고 채널 정보를 입력합니다.
3. 채널이 만들어지면 검색 허용을 **ON**으로 설정합니다.

**3단계: 카카오 i 오픈빌더에서 챗봇 만들기**

1. [카카오 i 오픈빌더](https://chatbot.kakao.com)에 접속합니다.
2. **챗봇 만들기**를 클릭하여 새 챗봇을 생성합니다.
3. 좌측 메뉴 **설정 → 채널 연결**에서 2단계에서 만든 채널을 연결합니다.

**4단계: 스킬(Skill) 등록**

1. 좌측 메뉴 **스킬 → 스킬 관리**로 이동합니다.
2. **스킬 등록**을 클릭합니다.
3. 스킬 URL에 아래 주소를 입력합니다:

```
https://RemoteAGT.twinverse.org/api/kakao/skill
```

4. 등록한 스킬을 각 블록(상태, 컨테이너, 프로젝트 등)에 연결합니다.

**5단계: 서버에 설정 적용**

`.env` 파일에 REST API 키를 입력합니다:

```
KAKAO_REST_API_KEY=여기에_REST_API_키_붙여넣기
```

**6단계: 서버 재시작**

```bash
pm2 restart remoteagt
```

이제 카카오톡에서 만든 채널을 친구추가하고 대화하면 RemoteAGT가 응답합니다! 🎉

## 📊 실시간 시스템 모니터링

RemoteAGT는 서버의 상태를 주기적으로 수집하여 웹 대시보드와 메신저 모두에서 확인할 수 있게 합니다.

### 모니터링되는 항목

| 항목 | 설명 | 시각화 |
|------|------|--------|
| CPU 사용률 | 현재 CPU가 얼마나 바쁜지 (%) | 프로그레스바 |
| 메모리 점유율 | RAM 사용량 / 전체 RAM (%) | 프로그레스바 |
| 디스크 용량 | SSD/HDD 사용량 / 전체 (%) | 프로그레스바 |
| 컨테이너 수 | 실행 중인 Docker 컨테이너 개수 | 숫자 |
| 컨테이너 상태 | 각 컨테이너의 Running/Stopped 여부 | 배지(Badge) |

> 💡 **초보자 안내**: CPU는 컴퓨터의 "두뇌 사용률", 메모리는 "책상 넓이", 디스크는 "서랍 용량"이라고 생각하시면 됩니다. 수치가 90%를 넘어가면 서버가 느려질 수 있으니 주의하세요!

## 🛡 역할 기반 접근 제어 (RBAC)

RemoteAGT는 3단계 사용자 권한 시스템을 제공합니다.

| 역할 | 권한 | 할 수 있는 것 |
|------|------|---------------|
| **User** (일반 사용자) | 조회 권한 | 대시보드 보기, 본인 작업 이력 확인 |
| **Admin** (관리자) | 조회 + 관리 권한 | 배포, 로그 조회, 컨테이너 관리 |
| **SuperAdmin** (최상위 관리자) | 모든 권한 | 사용자 관리, 역할 변경, 감사 로그, 관리자 통계 |

### 통합 인증 (Orbitron ↔ RemoteAGT)

Orbitron에서 가입한 계정으로 RemoteAGT에도 자동으로 로그인할 수 있습니다. 별도 회원가입이 필요 없으며, 동일한 이메일과 비밀번호를 사용합니다.

## 📜 활동 감사 로그 (Audit Log)

누가 언제 무엇을 했는지 자동으로 기록합니다.

기록되는 활동 예시:
- 사용자 로그인/로그아웃
- 프로젝트 배포 실행
- 컨테이너 로그 조회
- 사용자 역할 변경
- 설정 변경

> 💡 감사 로그는 보안 사고 대응과 팀 관리에 필수적인 기능입니다. "어제 밤에 누가 서버를 재배포했지?" 같은 질문에 즉시 답할 수 있습니다.
