# 백그라운드 워커 (Background Workers) 배포

사용자가 웹 브라우저를 통해 접속할 수 있는 어떠한 화면(UI)이나 외부 API 주소(URL Domain) 도 할당되지 않지만, **인터넷이라는 거대한 무대의 장막 뒤편에서 365일 조용히, 그리고 죽지 않고 묵묵히 돌아가야만 하는 프로세스**들이 있습니다.

Orbitron에서는 이를 **백그라운드 워커(Background Workers)**라고 부르며, 외부의 포트(Port) 접속 개방 없이 완벽히 고립된 컨테이너 환경에서 내부 로직만을 미친 듯이 연산해 내고 강제 유지(Keep-Alive) 시킵니다.

---

## 🛠 워커는 언제 사용하나요?

주로 다음과 같은 주기적 타스크(스케줄러)나 봇 프로그램들을 굴릴 때 요긴합니다. 

*   **Discord / 카카오톡 챗봇:** 사용자가 접속하는게 아니라, 봇 프로그램 자체가 메신저 서버에 24시간 연결을 유지하며 대기(Listen)하고 있는 유형의 코드.
*   **주기적 스케줄러 알고리즘 (Cron Job):** 매일 자정 정각 12시가 될 때마다 고객 데이터베이스를 일괄 업데이트하거나 요금을 청구 정산하는 스크립트. 
*   **이벤트 스트리밍 감지 (Queue 소비):** Kafka, RabbitMQ 등 메시지 큐의 큐(Queue) 통로에 가득 쌓인 밀린 작업량(ex. 이미지 리사이징 100건, 메일 발송 1000건)을 끊임없이 꺼내서 주구장창 처리하는 소비자(Consumer) 컨테이너.
*   **크롤러 (Web Scraper):** 하루 24시간 비트코인 거래소나 주식 사이트를 긁으시나요? 포트를 열어둘 필요가 전혀 없는 완벽한 백그라운드 작업입니다.

---

## 🚀 사용 가이드

백그라운드로 굴리기 위한 세팅은 일반 웹 서비스와 비슷하지만 결정적인 약간의 차이가 있습니다. 

### 워커 프로젝트 생성
1. 대시보드 우측 상단 `+ New Project` 모달에서 좌측 세 번째 탭인 **⚙️ 백그라운드 워커** 를 클릭해 진입합니다. 
2. 앞선 일반 배포와 똑같이 무한 구동시킬 봇이나 스케줄러의 ソ스 코드가 담긴 GitHub 레포지토리 주소를 쓰거나 ZIP 파일을 업로드합니다.
3. 배포 환경(예: Python, Node)을 고르고 생성을 누릅니다.

### ⚠️ 아주 중요한 차이점: `bind` 하지 마세요!
가장 주의해야 할 점은 당신의 워커 코드 내부에서 억지로 특정 포트를 할당(`server.listen(3000)`, `app.run(port=8080)` 등)하려고 들거나 **바인딩하면 안 됩니다.** 외부의 브라우저에서 들어오는 구멍 자체가 아예 시스템적으로 막혀 있기 때문에 무의미한 에러만 유발할 수 있습니다. 

그저 파이썬이면 파이썬 무한 루프 문법(`while True:`) 이나, 스케줄러 프레임워크 객체를 평범하게 띄워놓고 종료만 되지 않게 돌리시면 됩니다. 

*   **웹 서비스 컨테이너:** 앞단에 Cloudflare와 Nginx 라우터가 대기하며 사용자 트래픽 핑퐁을 중계해 주는 구멍 뚫린 환경.
*   **워커 컨테이너:** 그런 거 다 빼고, 오직 OS 속에서 CPU와 메모리 자원만 공급받으며 나 홀로 미친 듯이 도는 은둔형 환경.

## 봇이 강제종료(Crash) 되면 어쩌죠?

안심하세요! 여러분 코드 속의 버그(메모리 초과, 참조 에러 등) 때문에 봇이 한밤중에 혼자 강제 종료되거나 멈춰도 걱정 마세요. 
Orbitron 커널의 데몬 관리자 시스템이 컨테이너의 건강 상태 모니터링 심박수를 감지하다가, 프로세스가 비정상 종료되는 즉시 **단 1초 만에 알리지도 않고 새로운 컨테이너를 살려내어 워커를 부활**시킵니다. (Self-Healing 메커니즘 24시간 풀 가동)
