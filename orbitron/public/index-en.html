🚀 Deploying Regular Apps (GitHub / ZIP Upload)

▼

1. Deploying via GitHub Connection

1. After logging into the dashboard, click the + New Project button.
2. Enter your repository URL in the "GitHub" tab and turn on the "Auto Deploy" toggle.
3. Specify build commands such as npm install && npm run build and click create.
4. Once deployed, copy the Webhook URL from the right side of the dashboard and register it in your GitHub repository's Webhooks settings to enable automatic re-deployments on every Push.

2. Single Folder (ZIP) Upload Method

1. Select all files at the root of your project and prepare a compressed ZIP file. (Do not compress the outermost folder, drag the contents directly)
2. Drop the ZIP file into the dropzone in the "Regular App Upload" mode.
3. Leave the port empty, and the system will auto-assign an available port to run the container.

🎮 Setting up Unreal Engine Pixel Streaming

▼

Orbitron automatically generates a Linux-based Unreal Matchmaker deployment template.

1. Packaging Requirements

When building in the Unreal Engine editor, ensure the target platform is set to Linux > Server.

2. Dashboard Upload

1. Click Unreal Engine Deploy under the Dedicated Deployment section in the left sidebar.
2. Enter your game's English name. (e.g., mygame)
3. Compress the extracted Linux Server root folder into a ZIP file and upload it.
4. Once Vulkan tools are setup and GPU port mapping concludes in the background, a "🎮 Start Game" button will appear. Clicking it takes you to the external Web Matchmaker Client.

🐳 Deploying Multi-Container Compose Apps

▼

💡 Overview: A massive orchestration feature that deploys gigantic systems with multiple interlocking containers—frontend, backend servers, Redis cache, queue workers, etc.—all with a single Push.

✅ Use Case: When your Node.js app and real-time chatting Redis need to be bundled together. No need to deploy them separately; simply place a Compose file in your project, and they will run automatically on a unified network.

🛠 How to Use:

1. Create a docker-compose.yml file at the root of your project folder to define your container architecture.
2. Set the main traffic receiver by including web, app, or api in the service name and specify the port.
3. Link the repository and deploy from the dashboard.
4. Orbitron detects the Compose file, skips single-build, spins up the entire stack as background multi-containers, and automatically routes traffic to the main service.

🎲 Unity WebGL Publishing Guide

▼

1. Preparing the WebGL Build Folder

Compress the entire index.html, Build, and TemplateData folders generated from your Unity WebGL build into a single ZIP file.

2. One-Click Hosting

1. Click Unity WebGL Deploy from the left sidebar.
2. Upload the compressed file. Orbitron's internal provisioning logic runs a dedicated Nginx Alpine container specifically to process the pre-compressed WebAssembly metadata.
3. Renders blazingly fast in the browser without Unity MIME warnings.

💬 Linking AI Model Keys

▼

To use Orbitron's core "Auto Error Analysis & Chatbot" feature, you must register an AI API Key per project.

1. Go to Dashboard -> Click a Project -> Navigate to the Settings tab.
2. Select your desired model (e.g., Claude 4.6 series or Gemini 2.5/3.0 series) from the 🤖 AI Error Analysis Model Setting dropdown.
3. Click the Get Key button to obtain a free API Key from the Anthropic Console or Google AI Studio.
4. Save the Key. The system will now read the logs, deduce causes, and prompt solutions whenever deployments fail. You can also converse normally from the AI Assistant tab in the sidebar.
